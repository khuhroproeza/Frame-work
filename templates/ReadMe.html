  4. SVM

Datasets

  1. Bohren

    2. Drilling Dataset from IFW. Data recorded from two different types of run. Normal run classified as 0, (drilling Process under normal conditions) Abnormal run classified as 1 (drilling Process with Chatter occurance)

  2. HPC Performance

    3. Data tken from target system called "D.A.V.I.D.E", an energy efficient supercomputer developed by E4 Computer Engineering and hosted by CINECA in Bologna, Italy. Data aggregated in 5-minutes long intervals. There are around 166 metrics (our features), i.e. core loads, temperatures, fan speed, power consumptions, etc. Normal Run classified as 0. Forced Anomalies related to the performance of the HPC classified as 1

  3. HPC Powersave

    4. Another branch of HPC dataset, where forced anomalies are focused on powersaving. Normal run classified as 0 and forced anomalies as 1

  4. Conditioning

    5. This dataset contains a timestamp and 25 signals of 8 different components. For example, L1 could be the current of a stepper motor and L2 the speed of that stepper motor. this data shows the process of a whole production plant. The process was run till failure. Thus intial readings of the dataset are classified as 0 and end failure classified as 1 (last 500 readings to be certain as per the author of the dataset).

  5. TEP

    6. The Tennessee Eastman process is a typical industrial process that consists of five main process units: a two-phase reactor where an exothermic reaction occurs, a separator, a stripper, a compressor, and a mixer. This is a nonlinear open-loop unstable process that has been used in many studies as a case study for plant-wide control, statistical process monitoring, sensor fault detection, and identification of data-driven network models. 0 Classifies Normal 1 classifies Abnormal data

Running the Program

  1. Editing Bash file named "exprun.sh"
  2. Setting input argument to either Option=('Show')or Option=('Train')
  3. Setting in number of Runs to perform
  4. Running the bash file from the main folder directory; eg: "./exprun.sh" via terminal

Framework Setup

  1. Framework
    1. Converters

      2. Contains Converter python file to divide data as per the requirement of the aloggorithm

    2. Datasets

      3. Contains all the datasets

    3. Measures

      4. Python functions to compute the measures used in the framework

    4. Model

      5. Contains all the algorithms used in the framework

    5. savedresult

      6. Contains all the saved results as pkl file saving, measures and names of dataset + algorithm

    6. static

      7. Contains CSS file for Flask Output Result html page

    7. Templates

      8. Contains all the html pages used in the framework

  2. Exprun.sh

    3. Main Bash file to start the program

  3. Flaskblog.py

    4. Flask file to output and render all the pages

  4. Main.py

    5. Main file which files all the algorithms and saves the pickle files

  5. Run.py

    6. Run file which accepts arguements from the bash file to either run or show framework

  6. SqlCreater

    7. Functions to save result as SQL and to create different dictionaries for the use of flask page rendering
